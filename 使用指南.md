# 服务器探针面板 — 使用指南

本指南配合 [README.md](README.md) 使用，侧重实操流程。

## 首次部署流程

> **两个组件，分开部署**：
> - **面板**（React 前端）：在你的电脑或面板服务器上运行
> - **Agent**（Node.js）：在**每台被监控的服务器**上运行
> - `scripts/deploy.sh` **仅用于部署 Agent**

### 第一步：启动面板

```bash
git clone https://github.com/xcicvas/server-monitor.git
cd server-monitor
npm install
npm run dev     # 开发模式，浏览器访问 http://localhost:5173
# 或
npm run build  # 生产构建，产物在 dist/
```

### 第二步：在被监控服务器上运行 Agent

**方式 A：一键部署（推荐，无需克隆整个项目）**

```bash
# 只下载部署脚本
curl -fsSL https://raw.githubusercontent.com/xcicvas/server-monitor/main/scripts/deploy.sh -o deploy.sh
chmod +x deploy.sh

# 全自动部署（自动生成密钥和随机密码）
sudo ./deploy.sh
```

**方式 B：本地直接运行**

```bash
git clone https://github.com/xcicvas/server-monitor.git
cd server-monitor
npm install

# 启动 Agent（首次运行自动创建 admin/admin 账号）
node api/agent.js
```

Agent 启动后会显示服务地址和管理员凭据。**首次登录后请立即修改密码**。

### 第三步：在面板中添加服务器

1. 浏览器打开面板（默认 http://localhost:5173）
2. 点击「添加服务器」
3. 填写：
   - **服务器名称**：如 `生产环境-北京`
   - **Agent 地址**：如 `http://192.168.1.100:7001`（必须包含协议头）
   - **用户名**：`admin`
   - **密码**：部署 Agent 时设置的密码
4. 点击「探测」确认连接，然后「添加」

添加成功后卡片立即显示该服务器的实时指标。

### 第四步（必做）：修改默认密码

```bash
# 1. 登录获取 JWT
TOKEN=$(curl -s -X POST http://AgentIP:7001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"默认密码"}' | jq -r '.token')

# 2. 修改密码
curl -X POST http://AgentIP:7001/api/auth/change-password \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"oldPassword":"默认密码","newPassword":"新密码（至少8位）"}'
```

## 远程服务器一键部署

> **说明：`deploy.sh` 只部署 Agent。**

```bash
# 只下载部署脚本
curl -fsSL https://raw.githubusercontent.com/xcicvas/server-monitor/main/scripts/deploy.sh -o deploy.sh
chmod +x deploy.sh

# 运行（自动生成密钥和随机密码）
sudo ./deploy.sh

# 自定义参数
sudo ./deploy.sh \
  -p 7001 \
  -j my_secret_key \
  -u admin \
  -w MyStr0ngPass \
  -o http://面板地址.com
```

## Docker 部署

```bash
git clone https://github.com/xcicvas/server-monitor.git
cd server-monitor

# 启动 Agent + 面板（docker-compose.yml 中配置）
docker-compose up -d

# 查看日志
docker-compose logs -f agent
```

## 多台服务器管理

所有服务器用**同一套账号密码**即可。面板中添加多台服务器时，用户名密码填同一套，各服务器独立运行互不影响。

如需不同密码，在各服务器上设置不同的 `ADMIN_PASSWORD` 环境变量。

## 查看指标（HTTP API）

```bash
# 获取 JWT token
TOKEN=$(curl -s -X POST http://localhost:7001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"你的密码"}' | jq -r '.token')

# 获取实时指标
curl http://localhost:7001/api/metrics \
  -H "Authorization: Bearer $TOKEN"

# 获取进程列表
curl http://localhost:7001/api/processes \
  -H "Authorization: Bearer $TOKEN"

# 获取历史数据（最近30条）
curl "http://localhost:7001/api/history?limit=30" \
  -H "Authorization: Bearer $TOKEN"

# 健康检查（无需认证）
curl http://localhost:7001/api/health
```

## 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `AGENT_PORT` | `7001` | Agent 监听端口 |
| `JWT_SECRET` | `change-me-in-production` | JWT 签名密钥，**生产必改** |
| `JWT_EXPIRY` | `24h` | 令牌过期时间 |
| `ADMIN_USERNAME` | `admin` | 首次部署的管理员用户名 |
| `ADMIN_PASSWORD` | `admin` | 首次部署的管理员密码，**部署后必改** |
| `ALLOWED_ORIGINS` | 不限制 | 允许的来源域名 |
| `ALLOW_REGISTER` | `false` | 是否允许注册 |

## 常见问题

**Agent 占用多少资源？**
CPU 几乎为零，内存约 30-50MB。

**面板显示离线？**
执行 `curl http://服务器IP:7001/api/health` 确认 Agent 正常运行。检查防火墙和云服务商安全组是否开放了 7001 端口。

**注册接口返回 403？**
正常。注册默认关闭。如需创建更多账号，在启动时设置 `ALLOW_REGISTER=true`。

**数据存在哪里？**
面板配置在浏览器 localStorage，Agent 数据在 `./data/` 目录。数据完全私有，不上传到任何服务器。

**如何导出配置？**
点击面板右上角导出菜单，可导出 JSON 备份。换设备后导入即可恢复所有服务器配置。
