# 服务器监控面板 - 使用指南

## 系统架构

本系统由两部分组成：

```
┌─────────────────────────────────────────────────────────┐
│                    监控面板 (浏览器)                       │
│                   http://localhost:5173                   │
│  · 添加/删除服务器                                        │
│  · 定时轮询各 Agent 采集数据                              │
│  · 可视化展示 CPU/内存/磁盘/网络                          │
└───────────────────────┬─────────────────────────────────┘
                        │ HTTP (每 N 秒轮询)
                        ▼
┌─────────────────────────────────────────────────────────┐
│                   Agent (被监控服务器)                    │
│               node api/agent.js                          │
│  · 读取本机系统指标                                       │
│  · 提供 /api/metrics HTTP 接口                           │
└─────────────────────────────────────────────────────────┘
```

---

## 快速开始

### 第一步：在一台服务器上运行 Agent

在被监控的服务器上执行：

```bash
# 进入项目目录
cd /path/to/project

# 启动 Agent（默认端口 7001）
node api/agent.js
```

Agent 启动后会显示：

```
╔══════════════════════════════════════════════════════════╗
║   服务器监控 Agent 已启动                                   ║
║   服务地址:   http://localhost:7001                       ║
║   指标接口:   http://localhost:7001/api/metrics           ║
╚══════════════════════════════════════════════════════════╝
```

**自定义端口：**

```bash
AGENT_PORT=8000 node api/agent.js
```

### 第二步：在面板添加服务器

1. 打开监控面板：http://localhost:5173
2. 点击右上角「**添加服务器**」
3. 填写表单：
   - **服务器名称**：自定义名称，如 `生产环境-北京`
   - **Agent 地址**：`http://服务器IP:7001`
   - **刷新间隔**：默认 5 秒（范围 1-300 秒）
4. 点击「**添加并开始监控**」

添加成功后，卡片会立即开始显示该服务器的实时指标。

---

## 指标说明

| 指标 | 说明 | 颜色规则 |
|------|------|----------|
| **CPU** | 处理器使用率 (%) | 绿 < 60% · 橙 60-80% · 红 > 80% |
| **Memory** | 内存使用率 (%) + 已用/总量 | 同上 |
| **Disk** | 磁盘使用率 (%) + 已用/总量 | 同上 |
| **Network** | 网络接口数 + 收发速率 | 固定紫色 |
| **Uptime** | 服务器连续运行时间 | — |

## 常见问题

### Q: 添加服务器时报"地址格式不正确"

确保填入的是完整的 HTTP 地址，例如：
- ✅ `http://192.168.1.100:7001`
- ✅ `http://example.com:7001`
- ❌ `192.168.1.100:7001`（缺少协议头）
- ❌ `example.com`（缺少端口）

### Q: 卡片显示"down"，无法连接

1. 确认 Agent 进程正在运行：`curl http://服务器IP:7001/api/health`
2. 检查防火墙是否放行了 Agent 端口（如 7001）
3. 如果是云服务器，确认安全组规则已开放端口

### Q: 面板刷新频率太低/太高

添加服务器时可调整「刷新间隔」，范围 1-300 秒。建议：
- 常用服务器：5-10 秒
- 生产服务器：30-60 秒

### Q: 如何监控多台服务器？

重复「添加服务器」步骤即可。面板支持同时监控任意数量的服务器，各服务器独立轮询。

### Q: 数据存储在哪里？

所有数据（服务器列表、配置）均存储在浏览器 **localStorage** 中，不会上传到任何服务器。关闭浏览器后数据不会丢失。

### Q: Agent 支持哪些系统？

Agent 基于 Node.js 运行，理论上支持 Linux、macOS、Windows 等所有支持 Node.js 的系统。

## 命令参考

| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动前端面板（默认 5173） |
| `npm run agent` | 启动 Agent（默认 7001） |
| `AGENT_PORT=8000 node api/agent.js` | 自定义 Agent 端口 |
| `npm run build` | 构建生产版本 |

## Agent 接口

Agent 暴露出以下 HTTP 接口：

| 接口 | 说明 | 示例响应 |
|------|------|----------|
| `GET /api/metrics` | 轻量指标（推荐） | `{ data: { cpu, memory, disk, network } }` |
| `GET /api/system` | 完整系统快照 | 包含更多进程/负载信息 |
| `GET /api/health` | 健康检查 | `{ ok: true, status: "online" }` |
